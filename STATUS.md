## Decisions Log
- 2026-05-11: **Reader/Editor relationship locked as toggle (one pane, not split, not WYSIWYG).** Single full-width pane with `[Edit] [Read]` segmented control. Read Along plays in Reader mode against rendered text; Edit mode keeps existing textarea behavior. Rationale: matches iA Writer + Obsidian reading-view conventions and gives the magazine register room to breathe at 720px max-width. Split-pane would dilute the "serious reading surface" framing; Typora-style WYSIWYG would collapse the two surfaces locked in the v1 scope. Spec at `docs/2026-05-11-reader-design.md`.
- 2026-05-11: **Read Along highlight is text-match, not source-position match.** When TTS playback advances to sentence N (in `parseUnits` indexing), the Reader highlight searches `#reader .lector-sentence` spans for one whose textContent matches. Robust to ordering differences. Edge case: duplicate sentences match the first occurrence; multi-sentence list items may drift. Deferred to v1.1.
- 2026-05-11: **Browser-mode parity** elected as the "web addition" direction over a separate Web Companion SKU or web-specific feature work. Tauri-shell `Browser ↗` button opens the OS default browser via `tauri.shell.open`; honors `feedback_standalone_apps.md` (still ships as a standalone app, customer just opts to view in their preferred browser). Rationale: owner specifically valued Grammarly + extensions support in browser mode. Marketing-site callout on uselector.app still pending.
- 2026-05-11: **In-app Help is a modal, not a separate docs page.** `docs/index.html` (old marketing prototype, pre-5/9 redesign) is no longer reachable from the product. Customers who need quick reference get a centered modal; customers who need full docs get a link to uselector.app. `/help/` route still mounted in `server.py` as a fallback for deep-linking but the in-app button bypasses it.
- 2026-05-11: **Dev Manager command must use quoted-set form** `set "VAR=val"` for env vars. Bare `set FOO=1 && ...` includes the trailing space before `&&` in the value (becomes `"1 "`), failing membership checks against `("1","true","yes")`. Caught after ~30min debugging the TTS-preload-not-skipping behavior. Apply this discipline to any future processes.yaml entry with env chains.
- 2026-04-25: PyInstaller retired for Lector specifically. Empirical evidence: 8 iterations exposing lazy-import edge cases (qwen_tts stale, unittest stdlib exclusion broke torch.quantization, language_tags JSON missing, en_core_web_sm spaCy model missing, google.api_core exclusion broke F5, transformers.pipeline lazy-attr unresolved, transformers.pipelines submodule unresolved, server.py port-collision cascade fooled diagnosis). Rationale: PyInstaller's static analysis cannot see Python's dynamic import patterns; every customer release would ship with a runtime ImportError. Embedded Python + uv install pattern is the proven approach for ML apps (ComfyUI/A1111/Ollama/Open WebUI all use it). PyInstaller stays for Ibis (LLC-002) and TASS (LLC-001) where dep trees fit it.
- 2026-04-25: Mac build deferred to a later session. Tauri config will have both bundle.windows + bundle.macOS sections from the start so Mac is configuration-and-signing work, not architecture work, when that session resumes. Windows-first does NOT complicate Mac (validated cross-platform Python distribution path: python.org embeddable for Windows + python-build-standalone for Mac, both shipped via uv).
